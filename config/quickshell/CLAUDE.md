# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A [Quickshell](https://quickshell.org) desktop shell configuration, written in QML (Qt Quick). Quickshell renders Wayland desktop UI (bars, widgets, popups) from declarative QML and exposes system integration types (clocks, IPC, Hyprland, and more).

This directory is the source for `~/.config/quickshell`. The `dotter` setup in the parent `dotfiles` repo symlinks it there (`.dotter/global.toml`: `"config/quickshell" = "~/.config/quickshell"`), so edits here go straight to the deployed config.

## Running

- `qs` (alias for `quickshell`) runs the `default` config, `~/.config/quickshell/shell.qml`.
- `qs -p .` runs by path from this directory without the symlink.
- `shell.qml` sets `settings.watchFiles: true`, so a running instance hot-reloads on save. There is no build step.
- For LSP, `qmlls` finds this tree via `.qmlls.ini` as a project-root marker. Quickshell **generates** `.qmlls.ini` at runtime (a symlink into its VFS), so it's gitignored, not committed — before the first `qs` run on a fresh host it's absent and `qmlls` falls back to the `.git` root marker (see `nvim/lsp/qmlls.lua`).

## Structure

Three directories split the tree by role: `services/` (singletons), `components/` (reusable primitives), `modules/` (feature UI). `shell.qml` stays at the root because Quickshell loads it and nothing else. Cross-directory references use directory imports — from inside `modules/`, that means `import "../components"` and `import "../services"`; from the root, `import "modules"`.

- `shell.qml`: entry point, the `ShellRoot`. Quickshell loads this file and no other. Imports `"modules"`; per screen via `Variants`: `Wallpaper`, `Bar`, `Border`, `Osd`, `LauncherPanel`; single instances: `NotificationPanel`, `Lock`; plus the launcher `IpcHandler` and global launcher state.
- `modules/Bar.qml`: vertical left `PanelWindow` (layer-shell bar), one per screen via `Variants { model: Quickshell.screens }` in `shell.qml`. Takes `modelData` and binds it to `screen`. Hosts `Workspaces` (top), and at the bottom the `Tray`, `VpnIndicator` + `VolumeIndicator`, a stacked HH/MM clock, and the `NotifIndicator` (bell) last. Owns the slide-out `Drawer`s: tray menu, calendar (clock), and the notification center (bell).
- `modules/NotifIndicator.qml`: bar bell (tinted) with an active-count badge, dimmed when DND; click opens the notification center. `modules/NotificationCenter.qml`: swaync-style panel — header (DND toggle + Clear-all) over a scrollable list of `Notifs.history` cards (click a card to remove it). Lives in the bar's `centerDrawer`.
- `modules/Media.qml`: compact now-playing widget over `Players.active` (MPRIS) — cover art, title/artist, prev/play-pause/next; hidden when nothing plays. Reusable (used on the lock screen).
- `modules/VpnIndicator.qml`: bar VPN status from `Vpn` (dim when down, bright + green dot when up); click opens the launcher VPN menu (`qs ipc call launcher menu vpn`); hover shows a `Tooltip` with the active endpoint names. `modules/VolumeIndicator.qml`: bar volume from `Audio` — icon reflects level/mute, scroll changes volume (`WheelHandler`), click toggles mute; hover shows a `Tooltip` with the percentage. The bar clock is a `StateButton` that opens a `Calendar` in a `Drawer` (`calendarDrawer`).
- `modules/Wallpaper.qml`: desktop wallpaper, one `PanelWindow` per screen (`Variants` in `shell.qml`) on `WlrLayer.Background`, click-through, showing `Config.wallpaper`. Replaces swaybg — stop the external daemon so they don't stack.
- `modules/Osd.qml`: volume OSD — a passive (`modal: false`) left-bar notch `Drawer`, one per screen, that pops in on `Audio` volume/mute change and auto-hides; a `ready` flag gates out the startup binding settle. Shows the volume icon + a level bar + percentage.
- `modules/Calendar.qml`: self-contained Monday-first month grid (today highlighted, prev/next arrows page months) computed in JS; lives in the bar's `calendarDrawer`.
- `modules/Workspaces.qml`: niri workspace chips for one output (`output: bar.screen.name`). Click focuses the workspace via `Niri.focusWorkspace`.
- `modules/Tray.qml`: system tray `Row` over `SystemTray.items` (`Quickshell.Services.SystemTray`). Left click `activate()`s, right click emits `requestMenu(handle, x)`, which `Bar.qml` routes into the `Drawer` as a notch dropdown. Items use `StateLayer` for hover/press feedback.
- `modules/TrayMenu.qml`: renders a `SystemTrayItem.menu` handle (`QsMenuOpener.children` → `QsMenuEntry`) as clickable rows; lives inside a modal `Drawer`.
- `modules/Notifications.qml`: a `ListView` (not a positioner — so it can `remove`/`delayRemove`) of auto-expiring notification cards over `Notifs.list` (click a card to dismiss; ScriptModel-backed `Notifs.list` is incremental so `add`/`displaced` fire). On dismiss/expire a card uses the ListView `remove` transition (slides out to the right + fades) — it leaves layout at once, so the rest reflow up in a single `displaced` motion (no leftover `spacing` to jump) and the notch background follows the `contentHeight` step via the `Drawer`'s resize `Behavior`. New cards pop in (`add`). Card bindings guard `modelData` with `?.` because the delegate is kept alive through the remove transition while its `modelData` goes null (it's out of layout by then, so a momentary blank doesn't affect sizing). `modules/NotificationPanel.qml` hosts it as a non-modal `Drawer` (`edge: "top-right"`) — a corner notch out of the top-right, shoulders blending into both borders, growing downward as cards arrive — a single instance on the primary screen, shown while `Notifs.count > 0`, keeping the springy notch open/close animation.
- `modules/Border.qml`: rounded "screen border" frame for one output (caelestia's border without the SDF plugin). Three thin click-through strut `PanelWindow`s reserve `Config.border.thickness` on top/right/bottom (`ExclusionMode.Normal` + explicit `exclusiveZone`; the left is already reserved by the vertical bar) so tiled windows inset. A separate full-screen, click-through (`mask: Region {}`), `WlrLayer.Top` overlay (Top, not Bottom, so it stays static — niri slides background/bottom layer surfaces with the workspace but keeps top/overlay fixed; instantiated before `Bar` in `shell.qml` so the bar draws over its left strip) fills the border region with `Colours.base` and rounds the inner corners — a single `Shape`/`ShapePath` with `fillRule: ShapePath.OddEvenFill` (outer screen rectangle with square corners + inner rounded-rect subpath at `border.rounding` = a hole the desktop shows through). Geometry renderer (not `CurveRenderer`) for the hole, MSAA layer for smooth corners.
- `menus/`: shell scripts backing the launcher's action menus. Each script's `list` subcommand prints one **tab-separated** line per item: `<active 1|0>\t<label>\t<sublabel>\t<command>`, where `command` is run via `sh -c` (detached) on select. Deliberately dependency-free (no jq). `menus/vpn.sh` lists OpenVPN units + Tailscale profiles with toggle commands (ported from the old waybar `vpn-menu.sh`); it reads the OpenVPN endpoint names and the privileged helper (`vpn.openvpn[]`, `vpn.privileged`, `vpn.vpnCtl`) from `~/.config/dotfiles/config.json` (parsed with sed/grep, no jq), falling back to defaults if absent. Add a menu: drop a script here and register it in `services/Menus.qml`.
- `modules/Launcher.qml`: launcher content — a `SearchList` (with a `ListRow` delegate) plus the mode/menu logic on top. Two modes: **list** (default) shows the registered menus (`Menus`) on top then apps (`Apps.query`) in one list — both filter by the search text; and **menu** (drilled into a chosen menu) shows that menu's items, where selecting one runs its `command` then re-lists so the active dot refreshes. A result is a menu definition (has `argv`), an app (`DesktopEntry`), or a menu item, so `kindOf(m)` classifies each and `rowLabel`/`rowSub`/`rowIcon`/`rowDot` + `run` branch on kind (a `Process` + `StdioCollector` runs the menu script and parses the TSV; the `dot` shows the active state for menu items). `run(item)` (wired to both `SearchList.activated` and the row's `clicked`) enters a menu / launches an app / runs a command per kind. `startMenu` (set via IPC) opens straight into a menu. Esc backs out of a menu then closes (`SearchList.escaped`). All the list mechanics (ScriptModel, gliding highlight, ApplyRange scroll, no-transition snapping) live in `SearchList`. `modules/LauncherPanel.qml` wraps it in a keyboard-focused (`OnDemand`) modal `Drawer` (`edge: "bottom"`, horizontally centred), instantiated **per monitor** via `Variants` in `shell.qml`; the launcher content is in a `Loader` (`active: panel.active || panel.visible`) so hidden monitors' launchers cost nothing. Global state in `shell.qml` (`launcherOpen` + `launcherOutput`) plus a single `IpcHandler` (`target: "launcher"`, `toggle`/`open`/`close`/`menu(id)`) opens only the instance whose `screen.name === Niri.focusedOutput`, so the launcher appears on the focused monitor with a native keyboard grab (no layer surface ever moves between outputs). Each panel emits `closedByUser` (Esc / click-away) to reset the global state. niri keybinds: `qs ipc call launcher toggle` (apps), `qs ipc call launcher menu vpn` (straight into a menu). Note: the `IpcHandler` may not re-register functions on hot-reload — restart quickshell after adding/renaming an IPC function.
- `modules/Lock.qml`: custom session lock (`WlSessionLock` + `Quickshell.Services.Pam`), styled after qylock — a blurred wallpaper background (`layer.effect: MultiEffect` on the wallpaper `Image`), a big clock + date near the top (`Time.timeStr`/`Time.dateStr`), a centred avatar/name/password, and a `Media` now-playing widget at the bottom. One `WlSessionLock` auto-manages a `WlSessionLockSurface` per screen. PAM auth uses the bundled `assets/pam.d/passwd` (`pam_unix` via the setuid `unix_chkpwd`, so it works as a non-root process) — `PamContext { config: "passwd"; configDirectory: Quickshell.shellDir + "/assets/pam.d" }`; on submit `pendingPw` is set and `pam.start()` runs, `onResponseRequired` → `respond`, `onCompleted` → `PamResult.Success` unsets `locked`. Lock via `IpcHandler` (`target: "lock"`, `lock`/`unlock`/`isLocked`): `qs ipc call lock lock`. **Note:** the `unlock` IPC clears the lock *without* PAM auth — a deliberate recovery hatch, reachable only by processes already running as you (the session bus is per-user); don't expose the IPC socket beyond the user session. Auto-locks after `Config.lock.timeout` seconds of inactivity via an `IdleMonitor` (`respectInhibitors: true`, disabled while already locked; `timeout: 0` turns it off). **Safety:** if PAM is misconfigured or quickshell crashes while locked, the compositor keeps the session locked — recover from a TTY with `loginctl unlock-session` or by restarting quickshell. `Config.lock` holds `wallpaper`/`avatar`/`name`.
- `components/Drawer.qml`: **the notch popup pattern — every dropdown/drawer must use this.** A full-screen, transparent layer-shell `PanelWindow` (`ExclusionMode.Ignore` so it overlaps the bar instead of being shoved aside) that slides a panel out to the right of the vertical bar. The panel is a single `NotchShape`, one variant per `edge`: `"top"` is the original; `"bottom"` mirrors it in Y; `"left"` (default) rotates 90° CCW (out of the vertical bar); `"right"` mirrors that in X; `"top-right"` is a corner notch (flush top + right edges, concave shoulders at top-left and bottom-right, convex bottom-left). `top`/`bottom` are positioned along the screen by `anchorX`, `left`/`right` by `anchorY`, `top-right` pins to the corner. `align` places along the cross-axis: `"center"` (default) on the anchor, `"start"` pins the near side (top/left) at it, `"end"` the far side (bottom/right). The panel sits flush with the *inner* edge of its surface, offset from the screen edge by `barSize`, so the concave shoulders flare past it onto the desktop and stay visible — `barSize` defaults to the bar width, so popups emerging from the border (launcher, notifications) set it to `Config.border.thickness`. The panel lives inside a `clipper` Item that excludes the `barSize` strip at the emergence edge (`clip: true`), so the sliding panel is hidden behind the bar/border and appears to emerge from *under* it rather than over it. `modal: true` (default) grabs the screen and closes on click-away (menus); `modal: false` masks input to the panel so the rest stays click-through (passive popups). Drive it with `open()`/`close()` or bind `shown`.
- `components/NotchShape.qml`: the filled `Shape`/`ShapePath` boilerplate shared by every edge variant of `Drawer` (CurveRenderer, zero stroke, `fillColor`). Declare the outline as `PathLine`/`PathArc` children and the start via `startX`/`startY`; the per-edge path geometry lives in `Drawer.qml`.
- `components/StateLayer.qml`: reusable hover/press highlight (`MouseArea` + translucent rounded overlay) with a Material-style ripple — a soft radial-gradient disc (`Shape` + `RadialGradient`) expanding from the press point, clipped to a rounded-rect matching the parent's `radius`. `shapeMorph` lets the disc expand slightly past the bounds.
- `components/StateButton.qml`: clickable rounded surface — a `Rectangle` with a `StateLayer` baked in (ripple on top) and a `clicked(event)` signal. The interactive primitive behind tray icons, workspace chips, menu rows and notification cards; put content as children.
- `components/Icon.qml`: square `Image` preset (`PreserveAspectFit`, crisp `sourceSize`). Set `size` and `source`. Used for full-colour themed icons (tray/launcher app icons).
- `components/Glyph.qml`: a single Nerd Font icon glyph rendered as `Text` (`JetBrainsMono Nerd Font`), coloured directly via `Colours.*` — no recolouring/effects, so it's crisp and visible on the dark bar. Used by the bar status icons (VPN/volume/bell) and the OSD; set `text` to a glyph, e.g. `String.fromCodePoint(0xf0582)`.
- `components/SwapText.qml`: `Text` that animates on content change (caelestia's two-phase swap — the chosen prop accelerates out via `standardAccel`, the text swaps via `PropertyAction`, then decelerates back in via `standardDecel`, each phase `animateDuration/2`). Used by the bar clock. `animate: false` disables it.
- `components/Tooltip.qml`: hover tooltip as a `PopupWindow` that pops out to the right of `hostItem` (bar content can't render outside the narrow bar surface). Set `active` to the host's hover (`StateButton.containsMouse`) and `text`; appears after a short delay.
- `components/SearchField.qml`: rounded search `TextInput` box; exposes `text`/`placeholder` and forwards `up`/`down`/`accepted`/`escaped` as signals. Width set by the parent.
- `components/ListRow.qml`: a selectable list row built on `StateButton` — leading `Icon`, `label` + `sublabel`, and an optional trailing state `dot` (`-1` none / `0` hollow / `1` filled). Transparent so a list highlight shows through. The delegate preset for `SearchList`.
- `components/SearchList.qml`: **the reusable command-palette / menu / config-screen skeleton.** A `SearchField` over a `ListView` (`ScriptModel` so filtering keeps live delegates with no flash; gliding highlight via `highlightFollowsCurrentItem: false` + `Behavior on y`; `highlightRangeMode: ApplyRange` so ↑/↓ scroll). Provide `values` (an array) and a `delegate` (typically `ListRow`); handle `activated(item, index)` (Enter — mouse clicks are the delegate's own `clicked`) and `escaped`. `activate()`/`reset()` focus/clear the field. Build a new menu or config screen by composing this with `ListRow`.
- `components/Anim.qml` / `components/CAnim.qml`: `NumberAnimation` / `ColorAnimation` presets bound to the motion tokens. Set `curve` to any `Appearance.anim.curves.*` array and override `duration`.
- `services/`: QML singletons (`pragma Singleton`). Reactive state — `Time.qml` (wraps `SystemClock`), `Notifs.qml` (a `NotificationServer` daemon exposing `list` for popups, a kept `history` of snapshots for the notification center, and a `dnd` flag that suppresses popups), `Players.qml` (the active MPRIS player → `Media`), `Niri.qml` (reads `niri msg --json event-stream` via a `Process`/`SplitParser`), `Apps.qml` (indexes `DesktopEntries.applications` with a self-contained JS fuzzy `query`), `Menus.qml` (registry of launcher action menus → scripts in `menus/`), `Audio.qml` (default Pipewire sink volume/mute via `PwObjectTracker`), `Vpn.qml` (polls `menus/vpn.sh list` every 10s → `active` + active `names`). Tokens — `Colours.qml` (Catppuccin Mocha palette), `Appearance.qml` (motion: durations + bezier curves), `Config.qml` (layout/sizing/behaviour tunables). Singletons are not global; a consumer must `import "services"` (or `"../services"` from `modules/`) before referencing the type, e.g. `Time.timeStr`.
  - `Niri.qml` keeps `workspaces` as a *structural* list (changes only on add/remove/rename) and tracks live focus/active/urgent state in separate id-keyed maps (`focusedId`, `activeByOutput`, `urgentIds`). This is deliberate: workspace chip delegates stay alive across focus switches, so flipping focus animates the existing chips instead of rebuilding them.

## Conventions

- **Notch popups:** all dropdowns/drawers slide out to the right of the vertical left bar as a rounded "notch" via `components/Drawer.qml` (concave shoulders blending into the bar). Caelestia gets this from a C++ `Caelestia.Blobs` SDF plugin; we deliberately reproduce it in pure QML `Shape` instead. Match this style for any new popup.
- **Tokens, not literals:** colours come from `Colours.*` (Catppuccin Mocha), sizes/spacing/timeouts from `Config.*`, motion from `Appearance.anim.*`. Don't reintroduce inline hex or magic numbers — add a token instead.
- **Animations:** use `Anim`/`CAnim` with `Appearance.anim` tokens, never hand-rolled `NumberAnimation` durations. These mirror caelestia's Material 3 "expressive" motion (reproduced as bezier arrays, not the C++ plugin). Pick `spatial` curves (they overshoot) for movement/size/open-close, `effects` curves for opacity/colour, `emphasized`/`standard` for neutral transitions. Positioners (`Row`/`Column`) get `add`/`move` transitions so children pop in and slide on reflow.

## Dependencies / gotchas

- QML modules rely on Quickshell's auto-discovery and version-less imports (`import Quickshell`, `import QtQuick`).
- Keep everything self-contained in this config dir. Do not import or depend on Caelestia (or any other external shell) modules/configs. Existing `Caelestia.*` imports should be replaced with local equivalents.
- Only one process can own `org.freedesktop.Notifications`. `Notifs.qml` won't receive notifications while another daemon (e.g. `swaync`) is running; stop the other daemon and Quickshell takes over automatically.
